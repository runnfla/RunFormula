1.a. Краткое описание.

Модуль RunFormula предназначен для вычисления математических выражений, представленных
в текстовом виде, с использованием, при необходимости, программной логики.

Поддерживаемые типы данных:
  - целые числа в десятичном, шестнадцатеричном и двоичном форматах;
  - числа с плавающей запятой в десятичной и научной нотации;
  - комплексные числа и операции над ними;
  - интервалы и операции над ними;
  - строки и символы ASCII;

Арифметические и логические операции: + - * / or and xor not shl (<<) shr (>>) mod
div == <> < <= >= > & (объединение строк) ** (возведение в целую степень).
Поддерживаются переменные и инициализация переменных во время выполнения через
внешнюю функцию.
Есть набор встроенных функций и возможность регистрации и использования
дополнительных пользовательских функций.
Функции управления потоком выполнения: if() repeat() exit() result() continue() break()
Возможность определения и использования функций непосредственно в формуле (подпрограммы).
Поддержка директивы define.
Компиляция исходной формулы в байт-код для многократного выполнения.
Минимум зависимостей (только SysUtils в базовой конфигурации).

1.b. Интеграция RunFormula в проект.
  - Скопируйте все файлы из каталога RunFormula (runformula.pas и все файлы .inc)
в проект или в отдельный каталог;
  - Добавьте RunFormula в раздел uses секции interface или implementation.
Например, так:

  implementation
    uses RunFormula;

или, если используется отдельный каталог:

  implementation
    uses RunFormula in 'RunFormula/runformula.pas';

Самый простой способ использования RunFormula — вызвать RunFlaParse и передать ее
вывод функциям RunFlaExecStr или RunFlaExecVrt. Например:

  ShowMessage( RunFlaExecStr( RunFlaParse('9 * 3') ) ); // отображает 27

RunFlaParse компилирует исходную формулу в байт-код для выполнения функциями RunFlaExecStr
или RunFlaExecVrt. Результатом RunFlaExecStr является строка, а RunFlaExecVrt —
переменная типа Variant.
См. соответствующие разделы справки для получения дополнительной информации об использовании.

2. Синтаксис формулы.
2.a. Комментарии.
Комментарии в формуле могут быть заданы в фигурных скобках, либо после двойной
наклонной черты. Например:

 { текст комментария }
 // текст комментария

2.b. Целые числа.
Целое число может быть задано в десятичном, шестнадцатеричном, двоичном формате.
Например:

  X = 12345       // целое в десятичной форме
  X = 0xA12B      // целое в шестнадцатеричной форме
  X = 0a12bH      // целое в шестнадцатеричной форме
  X = 10101b      // целое в двоичной форме

Регистр символов не учитывается, за исключением префикса 0x.
При переполнении десятичное число преобразуется в вещественное, но для шестнадцатеричных
и двоичных чисел переполнение не проверяется.
При задании отрицательных чисел рекомендуется не отделять минус от числа:

  X = -999        // так рекомендуется
  X = -  999      // так не рекомендуется, но допускается

Следует быть осторожным при задании большого отрицательного шестнадцатеричного или
двоичного числа. Например:

  X = -0x8000000000000000    // результат  -9223372036854775808, целое
  X = - 0x8000000000000000   // результат -9.22337203685478E+18, вещественное

2.с. Вещественные числа.
Вещественное число может быть задано в виде десятичного числа с точкой или в виде
числа с порядком. Например:

  X = 12.34
  Y = 0.12
  X = 1E1
  Y = 0.2e-4
  Z = -23E+34

При задании отрицательных чисел также рекомендуется не отделять минус от числа.

2.d. Текстовые строки.
Текстовая строка может быть задана в кавычках или апострофах. Если строка задана в
кавычках, то в ней могут быть апострофы и наоборот. Например:

  X = '"A"' & "'s'"      // результат "A"'s'

Пустая строка может быть задана как "", '' или константой None.

2.e. Символы ASCII.
Символ ASCII может быть задан в виде строки, содержащей этот символ. В отличие от строки,
символ ASCII может участвовать в арифметических операциях как целое, соответствующее
его коду. Также может быть задан управляющий символ с помощью префикса "^". Например:

  X = "B" - "A"       // результат 1
  X = "B" & "A"       // результат BA
  X = "One" & "^M" & "^J" & "Two"   // вывод One и Two в 2 строки

2.f. Константы.
Имеются предопределенные константы:
  False   // 0 целое
  True    // 1 целое
  None    // пустая строка
  Pi      // 3.14159265358979323846
  E       // 2.71828182845904523536
  J       // мнимая 1
Регистр символов не учитывается.

2.g. Комплексные числа.
Для задания комплексного числа следует использовать выражение с константой j - мнимая 1,
или встроенную функцию Cplex(R, I). Например:

  X = 4+5*j          // результат (4,5)
  X = Cplex(2, 3)     // результат (2,3)

2.h. Интервалы.
Интервалы могут быть заданы с помощью функции Range(L, R). Например:
  X = Range(19, 23.5)      // результат [19:23.5]

2.i. Скобки.
Выражение может быть заключено в круглые скобки, значение "скобок" - это результат этого
выражения. В скобки может быть заключено несколько выражений, разделенных запятой или
точкой с запятой. Тогда выражения в скобках вычисляются последовательно слева направо,
и результат "скобок" - значение последнего выражения. Например:

  5 * (4 - 7)                     // результат -15
  5 * ( 21 , 21 + 1; 21 + 2 )     // результат 115

Скобки могут быть вложены друг в друга. Перед закрывающей круглой скобкой запятую или
точку с запятой ставить допускается, но не рекомендуется.

3.a. Математические операции.
В таблице перечислены поддерживаемые математические операции для разных типов значений.

                    Целое    Вещественное    Комплексное    Интервал   Символ ASCII   Строка

== <> < > <= >=       +           +               +             +         целое          +
+ - *                 +           +               +             +         целое          -
/                  веществ.       +               +             +        веществ.        -
or and xor div mod    +           -               -             -         целое          -
shl (<<) shr (>>) not (унарный)
- (унарный)           +           +               +             +         целое          -
**                 веществ.       +               +             +           -            -
&                  строка       строка          строка        строка        +            +

При переполнении целых в операциях + - * результат конвертируется в вещественный тип.
** - операция возведения в целую степень (положительную или отрицательную)
& - операция объединения строк

3.b. Приоритет операций.
В выражении математические операции выполняются в соответствии со следующим приоритетом:
  максимальный:    - not   // унарные
                   **
                   shl (<<) shr (>>) div mod
                   * / and
                   + - or xor
                   &
                   == <> < > <= >=
  минимальный:     =        // присваивание

3.c. Преобразование типов.
Тип значения может быть преобразован в соответствии со следующей таблицей. В ней также
указан тип Variant, используемый для передачи значений во внешние функции.

                    Целое    Вещественное    Комплексное    Интервал   Символ ASCII   Строка   Variant

Целое         -->     +           +             X+0*i         [X:X]          -          +        +
Вещественное  -->     -           +             X+0*i         [X:X]          -          +        +
Комплексное   -->     -           -              +              -            -        (R,I)      R
Интервал      -->     -         (A+B)/2      (A+B)/2+0*i        +            -        [A:B]      (A+B)/2
Символ ASCII  -->     +           -              -            [X:X]          +          +        +
Строка        -->     -           -              -              -            -          +        +
Variant       -->     +           +              -              -       if string[1]    +        NO

4.a. Переменные и операция присваивания.
В выражении могут использоваться переменные. В имени переменной допускаются буквы
латинского алфавита, цифры, символ точки (за исключением первого символа), символ
подчеркивания. Регистр символов не учитывается.
Значение переменной определяется с помощью операции присваивания:
  <переменная> = <выражение>
Перед использованием в выражении переменная должна быть определена с помощью оператора
присваивания. Результат операции присваивания равен присваиваемому значению. Например:

  X=2+4                     // результат 6
  abc=18/3                  // результат 6
  ( X=4, Y=6.1;  Z=X+Y )    // результат 10.1

4.b. Инициализация переменных во время вычисления.
Значение переменной может быть присвоено во время вычисления с помощью вызова внешней
глобальной функции следующего типа:

  function(constref Name:string; out Save:boolean):Variant;

В этой функции параметр Name - имя (в нижнем регистре) запрашиваемой переменной.
Параметр Save определяет, должно ли быть сохранено значение в переменной (True), либо
значение должно запрашиваться каждый раз (False). Переменная (при установленном значении
Save) всегда сохраняется в глобальном списке переменных.
Например, определим такую глобальную функцию:

  function MyRunFlaVar(constref Name:string; out Save:boolean):Variant;
  begin
    Result:=InputBox('', 'Get value for variable '+Name, '');
    Save:=false;
  end;

Укажем ее адрес в функции RunFlaExecStr/RunFlaExecVrt:

  ShowMessage( RunFlaExecStr(RunFlaParse('A & A'), Error, @MyRunFlaVar) );

О параметре Error смотрите раздел справки об обработке ошибок.
Результатом будет отображение строки, состоящей из двух объединеных введенных
пользователем строк.

Замечание. Параметр Name имеет корректное значение только во время выполнения функции
RunFlaExecStr/RunFlaExecVrt. Если значение Name необходимо и после, то его следует
скопировать в новую строку, например, так:  UniqueString(Name) или
MyName:=StrPas(PChar(Name));

5. Обработка ошибок.
5.a. Структура TRunFlaError.
Для получения информации об ошибке во время выполнения функций RunFlaParse и
RunFlaExecStr/RunFlaExecVrt служит структура TRunFlaError, определенная в файле
runflaerr.inc. В этой структуре Code - код ошибки из списка TRunFlaErrCode,
Position - позиция ошибки, начиная с 0, в исходной формуле. Переменная с типом
TRunFlaError должна быть передана в функции RunFlaParse и RunFlaExecStr/RunFlaExecVrt.
Функция RunFlaParse заполнит эту структуру информацией об ошибке. Если ошибки нет,
то Code будет присвоен код OK, Position - 0.
Функции RunFlaExecStr/RunFlaExecVrt заполняют структуру TRunFlaError при условии, что
в ней не содержится информация об ошибке (Code = OK).

5.b. Глобальная переменная RunFlaErrCode.
Код ошибки во время выполнения функций RunFlaParse, RunFlaExecStr, RunFlaExecVrt
и RunFlaFuncReg также записывается в глобальную переменную RunFlaErrCode, но при условии,
что в ней до этого не содержится информация об ошибке (RunFlaErrCode = OK).

5.c. Результаты функций RunFlaParse и RunFlaExecStr/RunFlaExecVrt при ошибке.
При возникновении ошибки в функциях RunFlaParse и RunFlaExecStr/RunFlaExecVrt их
результатом будет пустая строка. При передаче пустой строки в качестве параметра Fla
в функции RunFlaParse и RunFlaExecStr/RunFlaExecVrt их результатом также будет пустая
строка, но ошибка в этом случае не возникает.
Замечание: Предача непустого, корректного, но не содержашего формулы параметра Fla в
функцию RunFlaParse (например, содержашего только комментарий) эквивалентно
передаче константы None (пустая строка).

5.d. Формирование ошибки пользователем.
В функциях запроса значения переменной (TRunFlaVar) и дополнительных функциях, определенных
пользователем (TRunFlaFunc), есть возможность прервать вычисление и сформировать ошибку с
кодом из перечня TRunFlaErrCode. Для этого служит процедура RunFlaRaise. Например:

  function MySumFunc(const ParamCount:SizeInt; Context:pointer):Variant;
  begin
    if ParamCount<>2 then RunFlaRaise(ParamNumber);          // проверка числа параметров
    Result:=RunFlaParam(0-ParamCount, Context)+RunFlaParam(1-ParamCount, Context);
  end;

5.e. Текстовое сообщение об ошибке.
Для преобразования кода ошибки в текстовое сообщение служит список RunFlaErrorMsg,
определенный в файле runflamsg.inc. Например:

  procedure TForm1.ExecButtonClick(Sender: TObject);
  var Error : TRunFlaError;
  begin
    ShowMessage( RunFlaExecStr(RunFlaParse('9 * 3x', Error), Error) );
    ShowMessage( RunFlaErrorMsg[Error.Code] + ' at: ' + IntToStr( Error.Position ));
  end;

Этот список и файл может быть заменен пользователем.

5.f. Рекомендуется отключить генерацию exception в MemGet

6. Функции.
6.a. Предопределенные функции.
RunFormula имеет набор предопределенных функций, описанных в соответствующей справке.

6.b. Пользовательские функции.
Имеется возможность зарегистрировать и использовать дополнительные пользовательские функции.
Пользовательская функция должна быть глобальной и иметь следующий заголовок:

  function MyFunc(const ParamCount:SizeInt; Context:pointer):Variant;

В нем ParamCount - количество переданных параметров, Context - указатель, необходимый
для получения значения параметров. Тип результата пользовательской функции - Variant.

6.c. Регистрация пользовательской функции.
Пользовательская функция должна быть зарегистрирована с помощью функции RunFlaFuncReg.
В функции RunFlaFuncReg Name - строка с именем функции, обязательно сточными символами,
Func - указатель на пользовательскую функцию. При возникновении ошибки регистрации функция
RunFlaFuncReg возвращает код ошибки из списка TRunFlaErrCode, либо OK, если ошибки нет.
Также RunFlaFuncReg заностит код ошибки в глобальную переменную RunFlaErrCode, если ранее
в ней не было другого кода ошибки.
Регистрация требуется однократно после запуска программы до вызова функции RunFlaParse
(и, соответственно, RunFlaExecStr/RunFlaExecVrt). Допускается многократная регистрация,
регистрация с изменением адреса функции и перекрытие предопределенных функций.
В этом случае RunFlaFuncReg возвратит код ошибки FuncExist, но информация будет изменена.

6.d. Получение значений параметров.
Для получения значений параметров пользовательской функции служит функция RunFlaParam.
В ней Оffset вычисляется как <0-based номера требуемого параметра> - ParamCount, Context
должен быть передан из заголовка пользовательской функции.
Замечание: значение параметра (в том числе и в случае предопределенной функции) вычисляется
только при запросе его внутри самой функции.

6.e. Пример пользовательской функции.

  function MyIFFunc(const ParamCount:SizeInt; Context:pointer):Variant;
  begin
    if ParamCount<>3 then RunFlaRaise(ParamNumber);          // проверка числа параметров
    if RunFlaParam(0-ParamCount, Context)>0                  // тестируем первый параметр
      then Result:=RunFlaParam(1-ParamCount, Context)        // если >0 возвращаем второй
      else Result:=RunFlaParam(2-ParamCount, Context);       // если нет, то третий
  end;
                                                             // регистрация с проверкой ошибки
  if RunFlaFuncReg('myiffunc', @MyIFFunc)<>OK then ShowMessage('Registration Error');

  теперь если введем формулу

  ( X=10, Y=20, MyIfFunc(10, X+10, Y-20) )

  получим результат 20. При этом вычитание Y-20 выполнятся не будет.

7. Функции-подпрограммы, определенные в формуле.
7.a. Определение функции-подпрограммы.
В формуле можно использовать функции-подпрограммы, определенные непосредственно в самой формуле.
Например:

  (  func ABC(A, B, C);
       A+B+C
     endfunc;

     ABC(10, 20, 30)  )

Функция-подпрограмма должна быть определена перед ее использованием. Функция-подпрограмма может
быть определена в любом месте формулы, где допустимо указывать выражение, в том числе, и
внутри другой функции-подпрограммы, однако область видимости всех функций-подпрограмм -
глобальная, независимо от места их определения.
Функция-подпрограмма определяется как:

  func <Имя>( <формальные параметры через [,;]> )[,;]
    одно или несколько выражений, разделенных [,;]
    результат функции-подпрограммы - результат последнего выражения
  endfunc

Запятая или точка с запятой перед endfunc не допускается.

7.b. Локальные переменные в функции-подпрограмме.
В функции-подпрограмме создается своя локальная область переменных. Для получения значения
переменной или присваивания значения осуществляется поиск иерархический поиск переменной
от локальной области к глобальной. Для того, чтобы создать переменную (с возможностью
её инициализации) в локальной области без поиска, служит функция Local. Например:

   (  func ABC(A, B, C);
         local(X, Y=10, Z=A);
         X=2;
         X+Y+Z+B+C
      endfunc;

      ABC(1, 2, 3)   )

Формальные переменные также создаются в локальной области.

7.c. Параметры по-умолчанию.
В параметрах может быть указако значение по-умолчанию, которое используется, если оно
не передано при вызове. Например:

        (  func ABC(A, B=15, C=34+23);
              A+B+C
           endfunc;

           ABC(1)   )

8. Директивы.
8.a. Директива define.
Директива define <идентификатор> "текст" - заменяет в формуле указанный идентификатор на текст.
 В идентификаторе допускаются буквы латинского алфавита, цифры, символ точки (за исключением первого символа), символ подчеркивания.
  Регистр символов не учитывается. Перед использованием в формуле идентификатора он должен быть определен.
  В подставляемом тексте также могут использоваться идентификаторы других определений.
  Например:

  define MySum " 2 plus 7 "
  define plus  ' + '
  MySum                        // результат 9

9. Запуск байт-кода.
Результат функции RunFlaParse (байт-код) может быть сохранен, а функции
RunFlaExecStr/RunFlaExecVrt могут быть многократно вызваны с ним без предварительного
вызова RunFlaParse. При этом требуется соблюдение следующих условий:
  - пользовательские функции должны быть зарегистрированы в том объеме и том порядке,
  который был до вызова RunFlaParse
  - для получения информации об ошибке поле TRunFlaError.Code и глобальная переменная
  RunFlaErrCode должна быть  инициализирована как OK.
  - если изменился исходный код RunFormula, то RunFlaParse должна быть выполнена заново.
