1.a. Краткое описание.

Модуль RunFormula предназначен для вычисления математических выражений и физических
формул, представленных в текстовом виде, с поддержкой скриптового языка и обработкой
единиц измерения.

Поддерживаемые типы данных:
  - целые числа в десятичном, шестнадцатеричном и двоичном форматах;
  - числа с плавающей запятой в десятичной и научной нотации;
  - комплексные числа и операции над ними;
  - интервалы и операции над ними;
  - строки и символы ASCII;

Арифметические и логические операции: + - * / or and xor not shl (<<) shr (>>) mod (%)
div == <> < <= >= > & (объединение строк) ** (возведение в целую степень).
Поддерживаются переменные и их динамическая инициализация во время выполнения через
внешнюю функцию обратного вызова.
Есть базовый набор встроенных функций и возможность регистрации и использования
дополнительных пользовательских функций.
Функции управления потоком выполнения: if() repeat() exit() result() continue() break()
Возможность объявления и последующего использования функций непосредственно в тексте
формулы (подпрограммы).
Поддержка физических размерностей, их контроль во время выполнения операций,
автоматический расчет размерности результата и автоматическая конвертация значений из
одной системы единиц в другую.
Поддерживается директива define.
Компиляция исходной формулы в байт-код для многократного выполнения.
Минимум зависимостей (только SysUtils в базовой конфигурации).

1.b. Интеграция RunFormula в проект.

  - Скопируйте все файлы из каталога RunFormula (runformula.pas и все файлы .inc)
в проект или в отдельный каталог;
  - Добавьте RunFormula в раздел uses секции interface или implementation.
Например, так:

  implementation
    uses RunFormula;

или, если используется отдельный каталог:

  implementation
    uses RunFormula in 'RunFormula/runformula.pas';

либо укажите путь к каталогу RunFormula в соответствующем параметре компилятора.

Самый простой способ использования RunFormula — вызвать функцию RunFlaParse и передать
ее результат в функциии RunFlaExecStr или RunFlaExecVrt. Например:

  ShowMessage( RunFlaExecStr( RunFlaParse('9 * 3') ) ); // отображает 27

Функция RunFlaParse компилирует исходную формулу в байт-код для выполнения функциями
RunFlaExecStr или RunFlaExecVrt. Результатом функции RunFlaExecStr является строка,
а RunFlaExecVrt — переменная типа Variant.
См. соответствующие разделы справки для получения дополнительной информации.

2. Синтаксис скриптового языка формулы.
2.a. Формат языка. Комментарии.

Скриптовый язык использует свободный формат записи. Пробелы, табуляция, переносы строк
являются разделителями там, где это необходимо, а в остальных случаях игнорируются.
Не допускаются пробелы внутри идентификаторов, целых и вещественных чисел, конструкциях
"функция(", "массив[", "единица^порядок". Регистр символов не учитывается, за исключением
оговоренных случаев.
Комментарии могут быть записаны в фигурных скобках или после двойной наклонной черты.
Например:

 { текст комментария }
 // текст комментария

2.b. Целые числа.

Целое число может быть задано в десятичной, шестнадцатеричной и двоичной форме.
Например:

  X = 12345       // целое в десятичной форме
  X = 0xA12B      // целое в шестнадцатеричной форме
  X = 0a12bH      // целое в шестнадцатеричной форме
  X = 10101b      // целое в двоичной форме

Регистр символов не учитывается, за исключением префикса 0x.
При переполнении десятичные числа преобразуются в вещественные, но для шестнадцатеричных
и двоичных чисел переполнение не проверяется.
Допустимые диапазоны целых чисел:

  -9223372036854775807 ... 9223372036854775807 в 64-разрядных системах;
  -2147483647 ... 2147483647 в 32-разрядных системах;
  -32767 ... 32767 в 16-разрядных системах;

Минимальное системное отрицательное число (-9223372036854775808 в 64-разрядных,
-2147483647 в 32-разрядных, -32768 в 16-разрядных системах) может быть использовано,
однако в некоторых арифметических операциях (+ - * **) оно считается вне допустимого
диапазона и преобразуется в вещественное число.
Конструкции "-число" или "- число" интерпретируются как положительное число и унарный
минус (с соответствующим приоритетом операции). Если необходимо задать именно
отрицательное число, то следует использовать квадратные скобки. Например:

  X = - 3 ** 2       // результат -9
  X = -10 ** 2       // результат -100
  X = [- 3] ** 2     // результат 9
  X = [-10] ** 2     // результат 100

2.с. Вещественные числа.

Вещественное число может быть задано в виде десятичного числа с точкой или в виде
числа с порядком. Например:

  X = 12.34
  Y = 0.12
  X = 1E1
  Y = 0.2e-4
  Z = 23E+34

Как и для целых чисел, конструкции "-число" или "- число" интерпретируются как
положительное число и унарный минус (с соответствующим приоритетом операции). Если
необходимо задать отрицательное число, то следует использовать квадратные скобки.
Например:

  X = - 3.5 ** 2          // результат -12.25
  X = -10.2E20 ** 2       // результат -1.0404E42
  X = [- 3.5] ** 2        // результат 12.25
  X = [-10.2E20] ** 2     // результат 1.0404E42

2.d. Текстовые строки.

Текстовая строка может быть задана в кавычках или апострофах. Если строка задана в
кавычках, то в ней могут быть апострофы и наоборот. Например:

  X = '"A"' & "'s'"      // результат "A"'s'

Пустая строка может быть задана как "", '' или константой None.

2.e. Символы ASCII.

Символ ASCII может быть задан в виде строки, содержащей этот символ. В отличие от строки,
символ ASCII может участвовать в арифметических операциях как целое, соответствующее
его коду. Также может быть задан управляющий символ с помощью префикса "^". Например:

  X = "B" - "A"       // результат 1
  X = "B" & "A"       // результат BA
  X = "One" & "^M" & "^J" & "Two"   // вывод One и Two в 2 строки

2.f. Константы.

Имеются предопределенные константы:

  True    // -1 целое
  False   // 0 целое
  None    // пустая строка, значение не задано
  Pi      // 3.14159265358979323846
  E       // 2.71828182845904523536
  J       // мнимая 1

2.g. Комплексные числа.

Для задания комплексного числа можно использовать квадратные скобки, выражение с
константой j - мнимая 1, или встроенную функцию cplex(R, I). Например:

  X = [(-12.1, - 23.3)]      // результат (-12.1,-23.3)
  X = 4+5*j                  // результат (4,5)
  X = cplex(2, 3)            // результат (2,3)

2.h. Интервалы.

Интервал может быть задан с помощью квадратных скобок или встроенной функции range(L, R).
Например:

  X = [-1.1: 23]           // результат [-1.1:23]
  X = range(19, 23.5)      // результат [19:23.5]

Левая граница интервала должна быть меньше или равна правой.

2.i. Квадратные скобки.

Любой тип данных (за исключением массива) может быть задан с помощью квадратных скобок.
Например:

  X = [""]                                           // пустая строка, None
  X = ["M"] + ['^C']                                 // символы ASCII
  X = ["qwerty"] & ['asdfg']                         // строки, не рекомендуется
  X = [-0x10] + [ - 20.3 ]                           // целые и вещественные числа
  X = [ ( - 12.1 ,-12.45)]                           // комплексное число
  X = [ - 100:25 ]-[ "^A" : "^Z" ]+[-"A":200.4]      // интервалы

2.j. Операторные скобки.

Одно или несколько выражений может быть заключено в круглые скобки. Выражения в скобках
вычисляются последовательно слева направо. Результат вычисления "скобок" - это результат
последнего выражения. Выражения в скобках разделяются запятой или точкой с запятой.
Например:

  5 * (4 - 7)                     // результат -15
  5 * ( 21 , 21 + 1; 21 + 2 )     // результат 115
  5 * ( 21 + 1; 21 + 2; 100 )     // результат 500

Скобки c выражениями могут быть вложены друг в друга. Допускается ставить запятую или
точку с запятой перед закрывающей круглой скобкой.
Вместо круглых скобок с той же целью можно использовать ключевые слова "do" и "end".
Например:

  do
    X = 10;
    Y = 20;
    X + Y
  end

Если группа выражений открывалась круглой скобкой, то она должна быть и закрыта круглой
скобкой, и, наоборот, если группа открывалась словом "do", то она должна быть закрыта
словом "end".

3.a. Математические операции.

В таблице перечислены поддерживаемые математические операции для разных типов значений.

                               Целое  Вещественное  Комплексное  Интервал  Символ ASCII  Строка

== <> < > <= >=                  +         +             +           +       как целое      +
+ - *                            +         +             +           +       как целое      -
/                          вещест. рез-т   +             +           +     вещест. рез-т    -
or and xor div mod (%)
shl (<<) shr (>>) not            +         -             -           -       как целое      -
- унарный                        +         +             +           +       как целое      -
**                               +         +             +           +       как целое      -
&                              строка    строка        строка      строка        +          +

** - операция возведения в целую степень (положительную или отрицательную)
& - операция объединения строк
При переполнении целых в операциях + - * ** - (унарный) результат конвертируется
в вещественный тип.

3.b. Приоритет операций.

В выражении математические операции выполняются в соответствии со следующим приоритетом:

  максимальный:  **                                 // возведение в целую степень
                 - not                              // унарные
                 * / shl (<<) shr (>>) div mod (%)
                 + -
                 &                                  // объединение строк
                 == <> < > <= >=
                 and
                 or xor
  минимальный:   =                                  // присваивание

Операции присваивания (=) и возведения в степень (**) являются правоассоциативными
и выполняются последовательно справа налево.

3.c. Преобразование типов.

Тип значения может быть преобразован в соответствии со следующей таблицей. В ней также
указан тип Variant, используемый для передачи значений во внешние функции.

                    Целое  Вещественное  Комплексное  Интервал  Символ ASCII  Строка  Variant

Целое         -->     +         +           X+0*j       [X:X]        -           +       +
Вещественное  -->     -         +           X+0*j       [X:X]        -           +       +
Комплексное   -->     -         -             +           -          -         (R,I)     R
Интервал      -->     -      (L+R)/2     (L+R)/2+0*j      +          -         [L:R]  (L+R)/2
Символ ASCII  -->     +         +           X+0*j       [X:X]        +           +       +
Строка        -->     -         -             -           -          -           +       +
Variant       -->     +         +             -           -     if string[1]     +       -

4.a. Переменные и операция присваивания.

В выражениях могут использоваться переменные. В имени переменной допускаются буквы
латинского алфавита, цифры, точки (за исключением первого символа), подчеркивание.
Значение переменной задается с помощью операции присваивания:

  <переменная> = <выражение>

Перед использованием в выражении переменная должна быть определена с помощью оператора
присваивания. Результат операции присваивания равен присваиваемому значению. Например:

  X=2+4                     // результат 6
  abc=18/3                  // результат 6
  ( X=4, Y=6.1;  Z=X+Y )    // результат 10.1

Поскольку операция присваивания правоассоциативная, в одном выражении допускается
присвоение значения сразу нескольким переменным. Например:

  X = Y = Z = 3, X * Y * Z  // результат 27

4.b. Инициализация переменных во время вычисления.

Значение переменной может быть присвоено во время вычисления формулы с помощью обратного
вызова внешней глобальной функции следующего типа:

  function(constref Name:string; out Dim:SizeInt; out Save:boolean):Variant;

Параметры функции:
  Name - имя (в нижнем регистре) запрашиваемой переменной;
  Dim - закодированная размерность значения переменной (0 - если не используется);
  Save = True - сохранить значение переменной в глобальном списке переменных и далее
   не запрашивать ее;

Например, определим такую глобальную функцию:

  function MyRunFlaVar(constref Name:string; out Dim:SizeInt; out Save:boolean):Variant;
  begin
    Result:=InputBox('', 'Get value for variable '+Name, '');
    Dim:=0;
    Save:=false;
  end;

Укажем ее адрес в функции RunFlaExecStr/RunFlaExecVrt:

  ShowMessage( RunFlaExecStr(RunFlaParse('A & A'), Error, @MyRunFlaVar) );

О параметре Error смотрите раздел справки об обработке ошибок.
Результатом будет отображение строки, состоящей из двух объединенных введенных
пользователем строк.

Внимание! Параметр Name имеет корректное значение только во время выполнения функции
RunFlaExecStr/RunFlaExecVrt. Если значение Name необходимо и после, то его следует
скопировать в новую строку, например, так:

  S:=Name;
  UniqueString(S);

или

  S:=StrPas(PChar(Name));

5. Использование размерностей физических величин.
5.а. Единицы измерения.

Значению может быть задана физическая размерность, состоящая из базовых единиц измерения.
Диапазон порядков базовых единиц в величине - от -127 до 127. В 64-разрядных системах
допускается до 8 базовых единиц, в 32-разрядных - до 4-х, в 16-разрядных - до 2.
Также может быть неограниченное количество производных от них величин.

5.b. Синтаксис размерностей.

Размерности в формуле задаются в обратных кавычках. Для размерностей, единиц измерения,
приставок учитывается регистр символов. Порядок единиц задается после символа "^" без
пробелов. Порядок "^1" может не указываться. Отрицательный порядок задается через указание
единицы с положительным порядком после символа "/". Например:

  10`m`; 10`m^3`; 10`m/s`; 10 `m / s^2`; 10 `1/m^2`; 10 `m^2 kg`

У единиц измерения могут быть указаны приставки (без пробела). Например:

 10`m` + 100`cm`             // результат 11`m`
 10`m^2` + 1000`mm^2`        // результат 10.001`m^2`
 10`cm/us^2` + 100`m/ms^2`   // результат 100.1`Gm/s^2`

5.c. Присвоение размерностей.

Размерность может быть присвоена целому, вещественному, комплексному числу, интервалу,
символу ASCII. Размерность можно присвоить числу непосредственно в формуле, значению
переменной в выражении (но не самой переменной), выражению, результату функции.
Например:

  X = 10 `m/s` + 20`mm/us`                   // результат 20.01`km/s`
  X = 10; X`m` + X`mm`                       // результат 10.01`m`
  X = 10; range(X, X+1)`mm`+1`mm`            // результат [0.011:0.012]`m`
  X = 10; (X *2)`mm` + (X+2)`cm`             // 140`mm`

При присвоении размерности производится преобразование значения в соответствии с
параметрами единиц изменения, их порядком, приставками. Повторное назначение размерности
не допускается.
При использовании квадратных скобок размерность может быть указана как внутри скобок
(рекомендуется), так и снаружи. Например:

 [(10,20) `m/s`] == [(10,20)]`m/s`
 ["^A":"^C" `km/h`] == [1:3] `km/h`

6. Обработка ошибок.
6.a. Структура TRunFlaError.

Для получения информации об ошибке во время выполнения функций RunFlaParse и
RunFlaExecStr/RunFlaExecVrt служит структура TRunFlaError, определенная в файле
runflaerr.inc. В ней Code - код ошибки из списка TRunFlaErrCode, Position - позиция
ошибки, начиная с 0, в исходной формуле. Value - результат последней операции в виде
текстовой строки, которая заполняется функциями RunFlaExecStr/RunFlaExecVrt.
Переменная с типом TRunFlaError должна быть передана в функции RunFlaParse и
RunFlaExecStr/RunFlaExecVrt. Функция RunFlaParse заполнит эту структуру информацией
об ошибке. Если ошибки нет, то Code будет присвоен код OK, Position - 0.
Функции RunFlaExecStr/RunFlaExecVrt заполняют структуру TRunFlaError при условии, что
в ней не содержится информация об ошибке (Code = OK).

6.b. Глобальная переменная RunFlaErrCode.

Код ошибки во время выполнения функций RunFlaParse, RunFlaExecStr, RunFlaExecVrt
и RunFlaFuncReg также записывается в глобальную переменную RunFlaErrCode, но при условии,
что в ней до этого не содержится информация об ошибке (RunFlaErrCode = OK).

6.c. Результаты функций RunFlaParse и RunFlaExecStr/RunFlaExecVrt при ошибке.

При возникновении ошибки в функциях RunFlaParse и RunFlaExecStr/RunFlaExecVrt их
результатом будет пустая строка. При передаче пустой строки в качестве параметра Fla
в функции RunFlaParse и RunFlaExecStr/RunFlaExecVrt их результатом также будет пустая
строка, но ошибка в этом случае не возникает.
Замечание: Предача непустого, корректного, но не содержашего формулы параметра Fla в
функцию RunFlaParse (например, содержашего только комментарий) эквивалентно
передаче константы None (пустая строка).

6.d. Формирование ошибки пользователем.

В функциях запроса значения переменной (TRunFlaVar) и дополнительных функциях, определенных
пользователем (TRunFlaFunc), есть возможность прервать вычисление и сформировать ошибку с
кодом из перечня TRunFlaErrCode. Для этого служит процедура RunFlaRaise. Например:

  function MySumFunc(const ParamCount:SizeInt; out Dim:SizeInt; Context:pointer):Variant;
  begin
    if ParamCount<>2 then RunFlaRaise(ParamNumber);          // проверка числа параметров
    Dim:=0;
    Result:=RunFlaParam(0-ParamCount, Context)+RunFlaParam(1-ParamCount, Context);
  end;

6.e. Текстовое сообщение об ошибке.

Для преобразования кода ошибки в текстовое сообщение служит список RunFlaErrorMsg,
определенный в файле runflamsg.inc. Например:

  procedure TForm1.ExecButtonClick(Sender: TObject);
  var Error : TRunFlaError;
  begin
    ShowMessage( RunFlaExecStr(RunFlaParse('9 * 3x', Error), Error) );
    ShowMessage( RunFlaErrorMsg[Error.Code].ErrMsg + ' at: ' + IntToStr( Error.Position ) +
      ' "'+Error.Value+'"');
  end;

Этот список и файл может быть заменен пользователем.

6.f. Рекомендуется отключить генерацию exception в MemGet

7. Функции.
7.a. Предопределенные функции.

RunFormula имеет набор предопределенных функций, описанных в соответствующей справке.

7.b. Пользовательские функции.

Имеется возможность зарегистрировать и использовать дополнительные пользовательские функции.
Пользовательская функция должна быть глобальной и иметь следующий заголовок:

  function MyFunc(const ParamCount:SizeInt; out Dim:SizeInt; Context:pointer):Variant;

ParamCount - количество переданных параметров;
Dim - закодированная размерность значения переменной (0 - если не используется);
Context - указатель, необходимый для получения значения параметров.
Тип результата пользовательской функции - Variant.

7.c. Регистрация пользовательской функции.

Пользовательская функция должна быть зарегистрирована с помощью функции RunFlaFuncReg.
В функции RunFlaFuncReg Name - строка с именем функции, обязательно сточными символами,
Func - указатель на пользовательскую функцию. При возникновении ошибки регистрации функция
RunFlaFuncReg возвращает код ошибки из списка TRunFlaErrCode, либо OK, если ошибки нет.
Также RunFlaFuncReg заностит код ошибки в глобальную переменную RunFlaErrCode, если ранее
в ней не было другого кода ошибки.
Регистрация требуется однократно после запуска программы до вызова функции RunFlaParse
(и, соответственно, RunFlaExecStr/RunFlaExecVrt). Допускается многократная регистрация,
регистрация с изменением адреса функции и перекрытие предопределенных функций.
В этом случае RunFlaFuncReg возвратит код ошибки FuncExist, но информация будет изменена.

7.d. Получение значений параметров.

Для получения значений параметров пользовательской функции служит функция RunFlaParam.
В ней Оffset вычисляется как <0-based номер требуемого параметра> - ParamCount, Context
должен быть передан из заголовка пользовательской функции.
Замечание: значение параметра (в том числе и в случае предопределенной функции) вычисляется
только при запросе его внутри самой функции.

7.e. Пример пользовательской функции.

  function MyIFFunc(const ParamCount:SizeInt; out Dim:SizeInt; Context:pointer):Variant;
  begin
    if ParamCount<>3 then RunFlaRaise(ParamNumber);          // проверка числа параметров
    if RunFlaParam(-3, Dim, Context)>0                       // тестируем первый параметр
      then Result:=RunFlaParam(-2, Dim, Context)             // если >0 возвращаем второй
      else Result:=RunFlaParam(-1, Dim, Context);            // если нет, то третий
  end;
                                                             // регистрация с проверкой ошибки
  if RunFlaFuncReg('myiffunc', @MyIFFunc)<>OK then ShowMessage('Registration Error');

  теперь если введем формулу

  ( X=10, Y=20, MyIfFunc(10, X+10, Y-20) )

  получим результат 20. При этом вычитание Y-20 выполнятся не будет.

8. Функции-подпрограммы, определенные в формуле.
8.a. Определение функции-подпрограммы.

В формуле можно использовать функции-подпрограммы, определенные непосредственно в самой формуле.
Например:

  (  func ABC(A, B, C);
       A+B+C
     endfunc;

     ABC(10, 20, 30)  )

Функция-подпрограмма должна быть определена перед ее использованием. Функция-подпрограмма может
быть определена в любом месте формулы, где допустимо указывать выражение, в том числе, и
внутри другой функции-подпрограммы, однако область видимости всех функций-подпрограмм -
глобальная, независимо от места их определения.
Функция-подпрограмма определяется как:

  func <Имя>( <формальные параметры через [,;]> )[,;]
    одно или несколько выражений, разделенных [,;]
    результат функции-подпрограммы - результат последнего выражения
  endfunc

8.b. Локальные переменные в функции-подпрограмме.

В функции-подпрограмме создается своя локальная область переменных. Для получения значения
переменной или присваивания значения осуществляется поиск иерархический поиск переменной
от локальной области к глобальной. Для того, чтобы создать переменную (с возможностью
её инициализации) в локальной области без поиска, служит функция Local. Например:

   (  func ABC(A, B, C);
         local(X, Y=10, Z=A);
         X=2;
         X+Y+Z+B+C
      endfunc;

      ABC(1, 2, 3)   )

Формальные переменные также создаются в локальной области.

8.c. Параметры по-умолчанию.

В параметрах может быть указако значение по-умолчанию, которое используется, если оно
не передано при вызове. Например:

        (  func ABC(A, B=15, C=34+23);
              A+B+C
           endfunc;

           ABC(1)   )

9. Директивы.
9.a. Директива define.
Директива define <идентификатор> "текст" - заменяет в формуле указанный идентификатор на текст.
 В идентификаторе допускаются буквы латинского алфавита, цифры, символ точки (за исключением первого символа), символ подчеркивания.
  Регистр символов не учитывается. Перед использованием в формуле идентификатора он должен быть определен.
  В подставляемом тексте также могут использоваться идентификаторы других определений.
  Например:

  define MySum " 2 plus 7 "
  define plus  ' + '
  MySum                        // результат 9

10. Запуск байт-кода.

Результат функции RunFlaParse (байт-код) может быть сохранен, а функции
RunFlaExecStr/RunFlaExecVrt могут быть многократно вызваны с ним без предварительного
вызова RunFlaParse. При этом требуется соблюдение следующих условий:
  - пользовательские функции должны быть зарегистрированы в том объеме и том порядке,
  который был до вызова RunFlaParse
  - для получения информации об ошибке поле TRunFlaError.Code и глобальная переменная
  RunFlaErrCode должна быть  инициализирована как OK.
  - если изменился исходный код RunFormula, то RunFlaParse должна быть выполнена заново.
